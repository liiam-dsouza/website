## Terminal Foundation: Shell & Prompt

My terminal workflow is built on `zsh` (Z Shell), with Powerlevel10k as the prompt theme and Zinit as the plugin manager. This setup gives me a fast, expressive shell environment while keeping configuration modular and startup time under control.

Powerlevel10k provides a highly informative prompt, surfacing useful context such as git status, exit codes, and directory information without becoming visually overwhelming. Zinit handles plugin management and lazy-loading, allowing me to scale my shell configuration without sacrificing performance. This foundation shapes how all other tools behave and render.

## The Tools

The following tools form the core of my day-to-day terminal workflow. Each one solves a specific problem and integrates cleanly with the rest of the setup.

### 1. FZF - A Command-Line Fuzzy Finder

`fzf` is one of the most important tools in my terminal. It provides fast, flexible fuzzy searching over files, command history, git branches, and more.

I use `fzf` heavily in combination with other tools like `fd`, `ripgrep`, and `zoxide` to quickly narrow down results without breaking flow. Small custom keybindings make it easy to invoke in different contexts, turning search into an interactive, low-friction operation.

### 2. Bat - A `Cat` Clone with Syntax Highlighting

`bat` replaces `cat` in my workflow, adding syntax highlighting, line numbers, and git integration by default. It‚Äôs especially useful when previewing files during fuzzy searches or reviewing configuration files.

Paired with `fzf`, `bat` provides rich previews that make navigating large codebases significantly easier, without needing to open a full editor.

### 3. Ripgrep - Fast Search for Code & text

`ripgrep` (`rg`) is my go-to tool for searching through code and text. It‚Äôs extremely fast, respects `.gitignore` by default, and works well across large repositories.

I often combine `rg` with `fzf` to create interactive search workflows, allowing me to jump directly from a search result to a specific file or location.

### 4. fd - A Fast, User-Friendly Alternative to `find`

`fd` provides a simpler, more intuitive interface for finding files and directories. It has sensible defaults, better output formatting, and integrates cleanly with other tools.

In practice, `fd` is frequently used as a backend for `fzf`, enabling quick file discovery without memorising complex `find` flags.

### 5. Zoxide - A Smarter `cd` Command

`zoxide` replaces traditional directory navigation with a smarter, frequency-based approach. Instead of typing long paths, I can jump to commonly used directories with minimal input.

Over time, it learns my habits and becomes faster the more I use it, which pairs particularly well with a keyboard-driven workflow.

### 7. TMUX - Terminal Multiplexer

`tmux` allows me to manage multiple terminal sessions, windows, and panes within a single terminal instance. This is especially useful for long-running processes, remote sessions, or parallel tasks.

My configuration focuses on simple keybindings and predictable layouts, keeping tmux powerful without becoming overwhelming.

```conf title="~/.config/tmux/tmux.conf"

unbind C-b        # Unbind default prefix
set -g prefix C-s # Change prefix to Ctrl+s

set -g mouse on # Enable mouse support

bind r source-file ~/.config/tmux/tmux.conf \; display "Config reloaded!"

bind | split-window -h -c "#{pane_current_path}" # Create horizontal split
bind - split-window -v -c "#{pane_current_path}" # Create vertical split
```

## How They Fit Together

The real power of this setup comes from how these tools compose. `fd` and `ripgrep` feed into `fzf`, previews are handled by `bat`, navigation is accelerated by `zoxide`, and everything runs within a tmux-managed environment.

Rather than relying on a single ‚Äúdo everything‚Äù tool, this approach focuses on small, composable utilities that work well together and can be swapped out or extended over time.
